# Privacy Policy

## Overview

The Bazaar is designed to minimize personal data processing while using public blockchain infrastructure. Most protocol interactions are wallet-based and do not require account registration.

## Data We Process

### Wallet and On-Chain Data

When you connect a wallet, the interface may process:

- Public wallet addresses
- Transaction hashes
- Public smart contract interaction metadata

This data is public by nature on supported blockchains and cannot be removed by The Bazaar.

### Client-Side Usage Data

Depending on deployment configuration, the frontend may process limited technical telemetry such as:

- Browser type and version
- Device and viewport metadata
- Performance diagnostics and error traces

Any telemetry should avoid direct collection of sensitive personal data.

## Data We Do Not Custody

The Bazaar does not custody:

- Private keys
- Seed phrases
- Wallet credentials
- User funds

Wallet security remains fully under user control.

## Cookies and Storage

The interface may use local browser storage to preserve UI preferences (for example filters, theme state, or dismissed notices). These values are local to the user device unless explicitly transmitted by deployment analytics.

## Third-Party Services

The Bazaar may integrate third-party infrastructure providers, including:

- RPC and blockchain index services
- Wallet connectors
- Oracle and pricing providers

Those providers may process network metadata according to their own privacy policies.

## International Use and Compliance

Users are responsible for complying with local regulations related to blockchain, taxation, and digital assets in their jurisdiction.

## Data Retention

Public blockchain data is permanently retained by the network. Any off-chain telemetry retention period depends on deployment infrastructure policy.
